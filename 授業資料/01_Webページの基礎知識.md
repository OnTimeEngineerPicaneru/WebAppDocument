# Webページの基礎知識

> Webアプリ作成講座｜配布用教材 01
---

## 1. Webサイトは「相手に伝える」ためにある

Webデザインの目的は、見た目を派手にすることだけではありません。情報やサービスを、必要な人へ正しく、わかりやすく届けることが大切です。

作り始める前に、次の3つを考えましょう。

1. だれに見てほしいか（対象）
2. 何を伝えたいか（内容）
3. 見た人に何をしてほしいか（目的）

たとえば学校行事の案内なら、日時と場所をすぐ見つけられ、必要なら申し込みまで迷わず進めるデザインがよいでしょう。

---

## 2. Webサイトのおもな種類

Webサイトは目的によって分類できます。ただし、1つのサイトが複数の役割を持つこともあります。

| 種類 | おもな目的 | 例 |
| --- | --- | --- |
| コーポレートサイト | 会社や団体の情報を伝える | 会社概要、事業内容、採用情報 |
| プロモーションサイト | 商品、サービス、イベントを宣伝する | 新商品の特設サイト、イベント告知 |
| ポートフォリオサイト | 自分の作品や実績を紹介する | イラスト、写真、制作したアプリ |
| ECサイト | インターネット上で商品を売買する | Amazon、楽天市場、BOOTH |
| メディアサイト | 記事やニュースを継続して発信する | ニュースサイト、ブログ、技術記事サイト |
| SNS | 利用者どうしが投稿や交流をする | Instagram、Facebook |

「Webページ」は、ブラウザーに表示される1枚のページです。複数のWebページをひとまとまりにしたものを、一般に「Webサイト」と呼びます。

### 考えてみよう

学校の部活動を紹介するサイトは、どの種類に近いでしょうか。1つに決められない場合は、その理由も考えてみましょう。

## 3. ユーザビリティとアクセシビリティ

### ユーザビリティ

ユーザビリティとは、利用する人にとっての「使いやすさ」です。目的の情報を見つけやすい、操作に迷わない、待ち時間が短い、といったことが含まれます。

### アクセシビリティ

アクセシビリティとは、年齢や障害、利用する機器などにかかわらず、できるだけ多くの人が情報や機能を利用できることです。

たとえば画像に説明文を付けると、画像を見られない人も読み上げソフトで内容を知ることができます。色だけで正解・不正解を表さず、文字や記号も組み合わせることも大切です。

### 見やすく、使いやすくするチェックポイント

- **文字と背景を見分けやすくする**：薄い灰色の文字を白い背景に置かない
- **大切な情報を目立たせる**：文字の大きさ、太さ、余白などに差を付ける
- **レイアウトをそろえる**：ロゴ、メニュー、ボタンの位置や見た目をページ間で統一する
- **読みやすい文章にする**：結論を先に書き、短い文と見出しで整理する
- **操作の結果を予想できるようにする**：リンクやボタンを、押せるものだとわかる見た目にする
- **表示を軽くする**：大きすぎる画像や不要な動画を使いすぎない
- **キーボードでも使えるようにする**：マウスだけに頼らない

一般的な大きさの文字では、文字色と背景色のコントラスト比は **4.5:1以上** が目安です。配色を決めるときは、色の好みだけでなく読みやすさも確認しましょう。

### 広告について考える

広告はサイトを運営するための収入源になります。一方、広告が多すぎたり、本文を隠したり、閉じるボタンがわかりにくかったりすると、利用者は目的を達成しにくくなります。収益と使いやすさの両方を考えて配置する必要があります。

## 4. インターネットとWebの違い

**インターネット**は、世界中のコンピューターやネットワークをつなぎ、データをやり取りするための大きな仕組みです。

**Web（World Wide Web）**は、インターネットを利用する仕組みの1つです。リンクで結ばれたページなどを、ブラウザーで閲覧できます。インターネットではWebのほかに、メール、オンラインゲーム、ファイル転送なども利用されています。

つまり、次のような関係です。

```text
インターネット（通信の土台）
└─ Web、メール、オンラインゲームなど
```

## 5. Webページが表示される仕組み

Webページを見るときは、主に「Webクライアント」と「Webサーバー」が働きます。

- **Webクライアント**：Webページを要求し、受け取って表示する側。通常は端末上のWebブラウザー
- **Webサーバー**：要求を受け取り、HTML、CSS、画像などのデータを返すコンピューターやソフトウェア

```text
あなた
  ↓ URLを入力
ブラウザー（クライアント）
  ├─ 1. サーバーを探す（DNS）
  ├─ 2. ページを要求する（HTTPリクエスト） ──→ Webサーバー
  └─ 4. 受け取ったデータを組み立てて表示  ←── 3. データを返す（HTTPレスポンス）
```

### HTTPSとは

HTTPは、ブラウザーとサーバーが通信するときのルールです。HTTPSは通信内容を暗号化して、途中で読み取られたり書き換えられたりする危険を減らします。ただし、HTTPSだからといって、そのサイトの内容や販売者が必ず安全だとは限りません。

### キャッシュとは

ブラウザーは、受け取った画像やCSSなどを一時的に保存することがあります。これをキャッシュといいます。同じデータを再利用できれば、表示が速くなり、通信量も減らせます。保存するか、いつ再確認するかはサーバーからの指示などで決まります。

## 6. URLを分解してみよう

URL（Uniform Resource Locator）は、インターネット上の情報の場所を示す住所のようなものです。

例：`https://example.com/lesson/index.html?mode=easy#quiz`

| 部分 | 名前 | 意味 |
| --- | --- | --- |
| `https` | スキーム（プロトコル） | どの通信方法を使うか |
| `example.com` | ドメイン名 | 接続先を表す名前 |
| `/lesson/index.html` | パス | サーバー内の情報の場所 |
| `?mode=easy` | クエリ | サーバーへ渡す追加情報 |
| `#quiz` | フラグメント | ページ内の特定の位置 |

ドメイン名は登録によって重複しないよう管理されます。一方、1つのWebサイトには通常たくさんのURLがあり、同じ内容に複数のURLからアクセスできる場合もあります。

## 7. HTML・CSS・JavaScriptの役割

Webページ制作では、主に次の3つを使います。

| 技術 | 役割 | たとえると |
| --- | --- | --- |
| HTML | 内容と構造を表す | 建物の骨組み |
| CSS | 色、配置、文字サイズなどを整える | 内装や服装 |
| JavaScript | 操作に応じた動きや処理を加える | スイッチや仕掛け |

HTMLは **HyperText Markup Language** の略で、プログラミング言語ではなくマークアップ言語です。「これは見出し」「これは段落」のように、内容の意味と構造をタグで表します。

## 8. はじめてのHTML

新しいファイルを作り、`index.html` という名前で保存しましょう。

```html
<!doctype html>
<html lang="ja">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>わたしのおすすめ</title>
  </head>
  <body>
    <h1>わたしのおすすめ</h1>
    <p>このページでは、わたしの好きなものを紹介します。</p>

    <h2>おすすめの理由</h2>
    <p>楽しくて、何度でも挑戦できるからです。</p>
  </body>
</html>
```

保存したファイルをブラウザーで開き、タイトルと文章が表示されることを確認してください。

### コードの意味

| コード | 意味 |
| --- | --- |
| `<!doctype html>` | HTMLの現在の標準形式で書かれた文書だと示す |
| `<html lang="ja">` | 文書全体を囲み、主な言語が日本語だと示す |
| `<head>` | 画面に直接表示しない設定などをまとめる |
| `<meta charset="UTF-8">` | 文字コードをUTF-8にする |
| `<meta name="viewport" ...>` | スマートフォンなどの画面幅に合わせる |
| `<title>` | ブラウザーのタブなどに表示する題名 |
| `<body>` | 画面に表示する内容をまとめる |

> `lang` の日本語コードは `ja` です。国を表すコードの `jp` ではありません。

## 9. 要素・タグ・属性

次のHTMLを見てみましょう。

```html
<a href="https://example.com">サンプルサイトを見る</a>
```

- **要素**：開始タグから終了タグまでを含む部品全体
- **開始タグ**：`<a href="https://example.com">`
- **内容**：`サンプルサイトを見る`
- **終了タグ**：`</a>`
- **属性**：要素に追加情報を与えるもの。この例では `href`

### よく使う要素

#### 見出し

```html
<h1>ページ全体の見出し</h1>
<h2>章の見出し</h2>
```

`h1`から`h6`まであり、数字は文字の大きさだけでなく、見出しの階層と重要度を表します。見た目を変える目的だけで番号を選ばないようにしましょう。

#### 段落

```html
<p>これは1つの段落です。</p>
```

#### 画像

```html
<img src="images/cat.jpg" alt="窓辺で眠る白い猫">
```

`src`は画像の場所、`alt`は画像の代わりになる説明です。装飾だけの画像には `alt=""` を使う場合もあります。`img`要素に終了タグはありません。

#### リスト

```html
<ul>
  <li>りんご</li>
  <li>みかん</li>
</ul>

<ol>
  <li>ファイルを保存する</li>
  <li>ブラウザーで開く</li>
</ol>
```

`ul`は順番に意味がないリスト、`ol`は手順や順位など順番に意味があるリストです。各項目は`li`で囲みます。

#### リンク

```html
<a href="https://developer.mozilla.org/ja/">Web技術を調べる</a>
```

`href`に移動先のURLを指定します。「こちら」のような言葉だけでなく、リンク先がわかる文章にすると親切です。
